The Data Science Methodology put forward by John B. Rollins, a Data Scientist in IBM Analytics, is discussed here. 📌 Why Methodology? A Methodology gives the Data Scientist a framework for designing an AI Project. The framework helps the team decide on the methods, processes, and strategies employed to obtain the correct output. It is the best way to organise the entire project and finish it systematically without losing time and cost. The methodology is cyclical and iterative — each step refines the previous one.
Module 1 — Steps 1 & 2
From Problem to Approach
The first module establishes the foundation of the entire project. Without a clear problem definition and the right analytical approach, all subsequent work is misdirected. This module answers two critical questions: What is the problem? and How will we use data to solve it?

2.1.1 — Business Understanding

Question: What is the problem that you are trying to solve?

In this stage, we first understand the problem of the customer by asking questions and trying to comprehend what is exactly required. This is also known as Problem Scoping and Defining. With this understanding we can figure out the objectives that support the customer's goal.

🎯

5W1H Problem Canvas

Use the 5W1H framework (Who, What, Where, When, Why, How) to deeply understand the issue at every stage — especially Business Understanding and Data Requirements.

🤝

Stakeholder Engagement

Ask relevant questions and engage in discussions with all stakeholders to identify specific requirements and create a comprehensive list of business needs.

🎨

Design Thinking

This stage also involves using the DT (Design Thinking) Framework to empathise with users and reframe problems in human-centric ways.

📋

Output

A clear, written problem statement that defines what success looks like — the compass for the entire project team.

2.1.2 — Analytic Approach

Question: How can you use the data to answer the question?

When the business problem has been established clearly, the data scientist defines the analytical approach to solve the problem. This involves seeking clarification from stakeholders to pick the most appropriate path. The team asks key questions:

📊

Regression

Do I need to find how much or how many? Use regression when the target is a continuous number (price, temperature, sales).

🏷️

Classification

Which category does the data belong to? Use classification when predicting a class label (spam/not spam, disease/no disease).

🔵

Clustering

Can the data be grouped? Use clustering for unsupervised grouping of similar items (customer segments).

🚨

Anomaly Detection

Is there any unusual pattern in the data? Detect fraud, network intrusions, or equipment faults.

The Four Types of Data Analytics

To solve a particular problem, there are four main types of data analytics:

📈
Step 1

Descriptive Analytics

Summarises past data to understand what has happened. Uses graphs, charts, mean, median, mode, variance and standard deviation.

📌 Example: Average marks of students in an exam; analysing last year's sales data.
🔍
Step 2

Diagnostic Analytics

Understands why things happened using root cause analysis, hypothesis testing, and correlation analysis.

📌 Example: Why did company sales drop? Was it poor customer service or low product quality?
🔮
Step 3

Predictive Analytics

Uses past data to make predictions about future events using regression, classification, and clustering.

📌 Example: Forecast sales, demand, inventory, customer purchase patterns based on previous data.
💡
Step 4

Prescriptive Analytics

Recommends the best action to take to achieve the desired outcome using optimisation and decision analysis.

📌 Example: Design the right strategy to increase festival-season sales by optimising pricing and marketing.

Module 2 — Steps 3 & 4
From Requirements to Collection
Once the analytic approach is defined, the next question is: what data do we need, and where do we get it? This module covers identifying data requirements and systematically collecting the right data from the right sources.

2.1.3 — Data Requirements

Question: What are the data requirements?

The requirements of data are determined by the analytic approach chosen in the previous stage. The 5W1H questioning method can be employed here too. It is necessary to identify the data content, formats, and sources for initial data collection.

📌

Type of Data

Identify the types of data required — numbers (quantitative), words (qualitative), or images (visual/unstructured).

🗂️

Data Structure

Consider the structure: a table (structured), a text file, a database, or an unstructured format like images and audio.

🌐

Data Sources

Identify the sources from which data can be collected — primary sources (surveys, IoT) or secondary sources (Kaggle, WHO).

🧹

Preprocessing

Define any necessary cleaning or organisation steps required before analysis can begin — handling NaN, format conversion.

Three Types of Data

📋

Structured Data

Organised in tables with rows and columns. Has a predefined schema.

📌 Examples: Customer databases, Excel sheets, SQL tables, CSV files
🖼️

Unstructured Data

Has no predefined structure. Difficult to analyse without preprocessing.

📌 Examples: Social media posts, images, videos, audio, emails
📧

Semi-Structured Data

Has some organisational properties but is not fully structured in a table.

📌 Examples: Emails, XML files, JSON, HTML pages

2.1.4 — Data Collection

Question: What occurs during data collection?

Data collection is a systematic process of gathering observations or measurements. In this phase, data requirements are revised and decisions are made as to whether more or less data is needed. Today's high-performance database analytics enable data scientists to utilise large datasets.

🔬

Primary Data Source

Original, firsthand data collected via direct observation, surveys, interviews, IoT sensors, or experiments. Raw, unprocessed, and unbiased.

Examples: Marketing campaigns, feedback forms, IoT sensor data

📚

Secondary Data Source

Data already stored and ready for use. Collected from books, journals, websites, databases, and online repositories.

Examples: Kaggle, WHO, World Bank, data.gov, UNICEF, Google

💡 Revision of Data Collection: The Data Collection stage may be revisited after the Data Understanding stage, where gaps in the data are identified and strategies are developed to either collect additional data or make substitutions to ensure data completeness. DBAs and programmers often work together to extract data from both primary and secondary sources.

Methods of Secondary Data Collection

Online Methods

  • Social media data tracking
  • Web scraping
  • Satellite data tracking
  • Smart forms (online surveys)

Online Data Repositories

  • data.gov (US Government data)
  • World Bank Open Data
  • UNICEF data portal
  • Kaggle datasets
  • WHO (World Health Organization)
  • Google Dataset Search

Module 3 — Steps 5 & 6
From Understanding to Preparation
This is the most time-consuming module in Data Science. Data is rarely clean when first collected — it contains errors, missing values, duplicates, and irrelevant fields. This module covers evaluating data quality and transforming it into a usable form.

2.1.5 — Data Understanding

Question: Is the data collected representative of the problem to be solved?

Data Understanding encompasses all activities related to constructing the dataset. We check whether the data collected represents the problem to be solved. The relevance, comprehensiveness, and suitability of the data are evaluated using descriptive statistics and visualisation.

📊

Univariate Analysis

Analyse each feature individually — mean, median, mode, range, variance, standard deviation, percentiles.

🔗

Pairwise Correlation

Assess how pairs of variables relate to each other. Helps identify which features are most predictive of the target.

📉

Visualisation

Apply histograms, scatter plots, box plots and bar charts to understand distribution, outliers, and patterns in the data.

🔎

Quality Check

Identify invalid values, incorrect data types, missing entries, and duplicates before moving to preparation.

2.1.6 — Data Preparation

Question: What additional work is required to manipulate and work with the data?

This stage covers all activities to build the dataset for the modelling step. Data is transformed into a state where it is easier to work with. Data preparation is the most time-consuming step among all Data Science stages.

🧹

Data Cleaning

Deal with invalid or missing values (NaN, typos, wrong formats), remove duplicate entries, and assign correct data types.

🔗

Combine Data

Merge data from multiple sources — archives, tables, and platforms — into a single unified dataset for modelling.

⚙️

Transform Data

Convert data into meaningful input variables — encoding categorical data, normalisation, and scaling.

🛠️

Feature Engineering

Select, modify, or create new features from raw data to improve model performance. Example: Age of house = Current year − Year built.

Feature Engineering — House Price Example

Raw Data Available

  • Area of the house (in sq.ft)
  • Number of bedrooms
  • Year the house was built

New Engineered Features

  • Age of house = Current year − Year built
  • Price per sq.ft = Price of house ÷ Area
  • Rooms per bedroom = Total rooms ÷ Bedrooms

Why Feature Engineering?

  • Helps the model make more accurate predictions
  • Removes irrelevant or redundant features
  • Captures domain knowledge in variables
  • Improves both accuracy and training speed
Module 4 — Steps 7 & 8
From Modelling to Evaluation
With clean, prepared data in hand, the data scientist now builds and refines models. The modelling process is usually iterative — leading to adjustments in data preparation. Multiple algorithms are tested to find the best model for the Capstone Project.

2.1.7 — AI Modelling

Question: In what way can the data be visualised to get to the required answer?

Data Modelling focuses on developing models that are either Descriptive or Predictive. Data scientists can test multiple algorithms to identify the most suitable model for the Capstone Project.

📊

Descriptive Modelling

Summarises and understands the characteristics of a dataset without making predictions. Goal: describe data, not predict or decide.

🔮

Predictive Modelling

Uses statistical algorithms to identify patterns and predict future outcomes or values from historical data using regression, classification, etc.

🎯

Training Set

The data scientist uses a training set — historical data where outcomes are already known — to train and calibrate the model.

🔄

Iterative

The modelling process leads to adjustments back in Data Preparation. Multiple algorithms are tried until the best one is identified.

Descriptive Modelling — Techniques

Summary Statistics

  • Mean, Median, Mode
  • Standard Deviation, Variance
  • Range (highest − lowest)
  • Percentiles (quartiles Q1, Q2, Q3)

Visualisations

  • Bar charts — compare categories
  • Histograms — frequency distribution
  • Pie charts — proportion of parts
  • Box plots — spread and outliers
  • Scatter plots — two-variable relationships

Predictive Modelling — Techniques

📏

Regression

Predicts a continuous target value. Example: house price, exam score, temperature next week.

🏷️

Classification

Predicts which category/class something belongs to. Example: spam/not spam, disease type, cuisine type.

🔵

Time-Series Forecasting

Predicts values at future time points based on historical time-ordered data. Example: stock prices, weather.

🌳

Decision Tree

A tree-like model used for both classification and regression. Implemented in Python using scikit-learn. Libraries: pandas, numpy, sklearn.

2.1.8 — Evaluation

Question: Does the model used really answer the initial question, or does it need to be adjusted?

Evaluation is the process of assessing how well a model performs after training. It involves using test data to measure metrics like accuracy, precision, recall, or F1 score. This helps determine if the model is reliable before deployment.

🔬

Phase 1: Diagnostic Measures

Ensures the model is working as intended. For a predictive model, a decision tree can evaluate whether output aligns with initial design or needs adjustment.

📐

Phase 2: Statistical Significance

Applied to verify the model accurately processes and interprets data. Designed to avoid unnecessary second-guessing when the answer is revealed.

🧪

Testing Set

For a descriptive model, a testing set with known outcomes is applied, and the model is refined as needed based on results.

🔁

Goes Back to Modelling

If evaluation reveals issues, the team returns to the Modelling stage to adjust algorithms, parameters, or feature selection.

Module 5 — Steps 9 & 10
From Deployment to Feedback
The final module brings the model to real users and collects feedback to continuously improve it. The methodology is cyclical — feedback from real-world use feeds back into every earlier stage, making the entire process iterative and self-improving.

2.1.9 — Deployment

Question: How does the solution reach the hands of the user?

Deployment refers to the stage where the trained AI model is made available to users in real-world applications. Data scientists must make stakeholders familiar with the tool produced in different scenarios. Once the model is evaluated and the data scientist is confident it will work, it is deployed.

🌐

Web Application

Embed the model as a REST API (Flask/FastAPI in Python) and integrate it into a website so users can submit data and get predictions.

📱

Mobile App

Deploy using TensorFlow Lite or ONNX to run models directly on smartphones. E.g., a cuisine-identification app.

🧪

Limited Rollout

The model may first be rolled out to a limited group of users or in a test environment to build confidence before full deployment.

👥

Team Collaboration

Deploying into a live business process frequently requires additional internal teams, skills, and technology beyond the data scientist alone.

2.1.10 — Feedback

Question: Is the problem solved? Has the question been satisfactorily answered?

The last stage in the methodology is feedback. This includes results collected from deployment, feedback on the model's performance from users and clients, and observations from the deployed environment. This process continues until the model provides satisfactory results.

📋

User Feedback

Feedback from users helps refine the model and assess it for performance and impact. Collected via app ratings, surveys, and usage analytics.

🤖

Automated Refresh

Data Scientists may automate any or all of the feedback so that the model refresh process speeds up and can get quick improved results.

🔄

Cyclical Process

The process from modelling to feedback is highly iterative. Throughout the DS Methodology, each step sets the stage for the next.

✅

Success Criteria

The cycle continues until: the problem is solved, the business question is satisfactorily answered, and the stakeholders are satisfied.

Section 2.2
Model Validation
Evaluating the performance of a trained machine learning model is essential. Model Validation offers a systematic approach to measure its accuracy and reliability, providing insights into how well it generalises to new, unseen data. It is conducted post-Model Training using a testing dataset.
⭐

Enhances Model Quality

Systematic evaluation reveals weaknesses in the model, allowing targeted improvements before production deployment.

🛡️

Reduces Risk of Errors

Catches poor-performing models before they affect real business decisions or user-facing applications.

⚖️

Prevents Overfitting

A model that memorises training data performs poorly on new data. Validation ensures generalisation, not memorisation.

📉

Prevents Underfitting

Validation also catches models that are too simple to capture the underlying patterns — underfitting gives poor results everywhere.

2.2.1 — Train-Test Split

The train-test split is a technique for evaluating the performance of a machine learning algorithm. It can be used for classification or regression problems and any supervised learning algorithm. The procedure involves taking a dataset and dividing it into two subsets.

🏋️

Training Dataset

Used to fit/train the machine learning model. The model learns patterns and relationships from this data.

🧪

Test Dataset

Used to evaluate the fit machine learning model. Predictions are made and compared to expected values.

Common Train-Test Split Configurations

80% / 20% Split (Most Common)
Train — 80%
Test 20%
70% / 30% Split
Train — 70%

67% / 33% Split
Train — 67%
Test — 33%
📌 No Optimal Split: There is no single optimal split percentage. Choose based on: computational cost of training and evaluation, how well the training set represents the full population, and how well the test set represents unseen data. The train-test procedure is appropriate when there is a sufficiently large dataset available.

2.2.2 — K-Fold Cross Validation

Cross Validation is a technique used to evaluate a model's performance. It splits the data into multiple parts called folds. It trains on some folds and tests on others, repeating for all folds. In k-fold cross validation, we work with k subsets of the dataset.

Section 2.3
Model Performance — Evaluation Metrics
Evaluation metrics help assess the performance of a trained model on a test dataset, providing insights into its strengths and weaknesses. These metrics enable comparison of different models to select the best-performing one for a specific task. Classification and Regression problems use different metrics.

📊 Evaluation Metrics — Classification vs Regression

2.3.1 — Evaluation Metrics for Classification

1. Confusion Matrix

A Confusion Matrix is a table used to evaluate the performance of a classification model. It summarises predictions against actual outcomes. It creates an N×N matrix, where N = number of classes. For binary classification (N=2), it creates a 2×2 matrix.

True Negative
TP: Predicted Yes, Actually Yes
TN: Predicted No, Actually No
FP: Predicted Yes, Actually No (Type I Error)
FN: Predicted No, Actually Yes (Type II Error)

2. Precision, Recall, F1-Score & Accuracy — Formulas

Metric 1
Precision
TP / (TP + FP)
What proportion of predicted Positives is truly Positive? Should be as high as possible.
Metric 2
Recall
TP / (TP + FN)
What proportion of actual Positives is correctly classified? Also called Sensitivity.
Metric 3
F1-Score
2 × (Precision × Recall) / (Precision + Recall)
Harmonic mean of Precision and Recall. Perfect = 1, Total failure = 0. Low FP and low FN = good F1.
Metric 4
Accuracy
(TP + TN) / (TP + FP + FN + TN)
Number of correct predictions ÷ Total number of predictions. Overall performance measure.

2.3.2 — Evaluation Metrics for Regression

Regression Metric 1
MAE
Σ |y − ŷ| / n
Mean Absolute Error — sum of absolute differences between predictions and actual values. Value of 0 = perfect predictions.
Regression Metric 2
MSE
Σ (y − ŷ)² / n
Mean Squared Error — most commonly used. Mean of squared distances between target and predicted values. Penalises large errors more.
Regression Metric 3
RMSE
√ MSE
Root Mean Square Error — standard deviation of residuals. Preferred over MSE because it is in the same units as the target variable.

2.4.2 — Confusion Matrix — Calculate All Metrics

Given: TP = 35, TN = 50, FP = 10, FN = 5. Calculate Precision, Recall, F1-Score, and Accuracy.

confusion_matrix_calculations.txt
Given:
TP = 35   # True Positives
TN = 50   # True Negatives
FP = 10   # False Positives
FN =  5   # False Negatives

Precision = TP / (TP + FP)
         = 35 / (35 + 10) = 35/45 = 0.778 → 77.8%

Recall    = TP / (TP + FN)
         = 35 / (35 + 5)  = 35/40 = 0.875 → 87.5%

F1-Score  = 2 × (Precision × Recall) / (Precision + Recall)
         = 2 × (0.778 × 0.875) / (0.778 + 0.875)
         = 2 × 0.681 / 1.653  = 0.823 → 82.3%

Accuracy  = (TP + TN) / (TP + FP + FN + TN)
         = (35 + 50) / (35 + 10 + 5 + 50)
         = 85 / 100 = 0.85 → 85%

2.4.3 — Python Code to Evaluate a Linear Regression Model

Using the Salary vs. Years of Experience dataset (from Kaggle) — 30 rows, 2 columns. Demonstrates the complete pipeline: load → EDA → prepare → train → evaluate.

salary_linear_regression.py
import pandas as pd
import numpy as np
import matplotlib.pyplot as plt
import seaborn as sns
from sklearn.metrics import mean_squared_error
from sklearn.linear_model import LinearRegression
from sklearn.model_selection import train_test_split

# ── Load Dataset ──
df = pd.read_csv('/kaggle/input/random-salary-data/Salary_Data.csv', sep=',')
print(df.head())

# ── EDA ──
print(df.shape)         # (30, 2)
print(df.isnull().sum())
# YearsExperience    0
# Salary             0    ← No missing values

# ── Data Preparation ──
X = np.array(df['YearsExperience']).reshape(-1, 1)
Y = np.array(df['Salary']).reshape(-1, 1)
print(X.shape, Y.shape)  # (30,1) (30,1)

# ── Configure Train-Test Split (80% / 20%) ──
X_train, X_test, Y_train, Y_test = train_test_split(
    X, Y, test_size=0.2, shuffle=True, random_state=10)

# ── Fit the Model ──
model = LinearRegression()
model.fit(X_train, Y_train)

# ── Evaluate on Training Data ──
train_score = model.score(X_train, Y_train)
print(f"Training R² score: {train_score:.4f}")

# ── Evaluate on Testing Data ──
test_score = model.score(X_test, Y_test)
print(f"Testing R² score:  {test_score:.4f}")

# ── Calculate Mean Squared Error ──
Y_pred = model.predict(X_test)
mse = mean_squared_error(Y_test, Y_pred)
rmse = np.sqrt(mse)
print(f"Mean Squared Error (MSE):  {mse:.2f}")
print(f"Root Mean Squared Error:   {rmse:.2f}")

# ── Visualise: Scatter Plot + Regression Line ──
plt.figure(figsize=(8, 5))
plt.scatter(X_test, Y_test, color='steelblue', label='Actual', alpha=0.7)
plt.plot(X_test, Y_pred, color='red', linewidth=2, label='Predicted Line')
plt.xlabel('Years of Experience')
plt.ylabel('Salary')
plt.title('Salary Estimation — Linear Regression')
plt.legend()
plt.tight_layout()
plt.show()
💡 Base Reference: Dataset from Kaggle — Random Salary Data of Employees (Age Wise). This practical demonstrates the full Data Science Methodology pipeline: Data Collection → Data Understanding (EDA) → Data Preparation → Modelling (Train-Test Split + Linear Regression) → Evaluation (MSE, RMSE, R²).

Q1. How many steps in DS Methodology? Name them in order.
10 steps: (1) Business Understanding (2) Analytic Approach
(3) Data Requirements (4) Data Collection (5) Data Understanding
(6) Data Preparation (7) AI Modelling (8) Evaluation
(9) Deployment (10) Feedback

Q2. What is Feature Engineering?
The process of selecting, modifying, or creating new features (variables)
from raw data to improve the performance of machine learning models.
E.g., Age of house = Current year − Year built; Price/sqft = Price ÷ Area.

Q3. Explain different types of data sources with examples.
Primary: Original firsthand data — surveys, IoT sensors, feedback forms.
Secondary: Pre-existing data — Kaggle, WHO, World Bank, web scraping,
social media tracking. DBAs and programmers often work together to
extract from both sources.

Q4. Which step is related to constructing the dataset? Explain.
Data Understanding (Step 5). It encompasses all activities to build the
dataset, evaluating relevance, comprehensiveness, and suitability using
descriptive statistics (univariate analysis, pairwise correlation) and
visualisation (histograms, scatter plots).

Q5. Short note on Data Preparation steps.
1. Cleaning: handle missing/invalid values, remove duplicates, correct formats.
2. Combine: merge data from multiple sources (archives, tables, platforms).
3. Transform: convert to meaningful input variables (encoding, normalisation).
4. Feature Engineering: create new derived features to improve model accuracy.
It is the MOST time-consuming step in DS Methodology.

Q6. Difference: Descriptive vs Predictive Modelling.
Descriptive: Summarises current data without predictions. Goal: understand.
 Techniques: mean, charts, histograms, box plots.
Predictive: Uses historical data to predict future outcomes. Goal: forecast.
 Techniques: regression, classification, time-series forecasting.

Q7. Metrics for evaluating Classification models.
1. Confusion Matrix: 2×2 table of TP, TN, FP, FN
2. Precision = TP/(TP+FP) — what proportion of predicted positives are real
3. Recall = TP/(TP+FN) — what proportion of actual positives were caught
4. F1 = 2×(P×R)/(P+R) — harmonic mean; 1=perfect, 0=failure
5. Accuracy = (TP+TN)/(TP+FP+FN+TN)

Q8. Is Feedback a necessary step? Justify.
Yes. Feedback collects results from deployment, user reviews, and
observations from the live environment. Without feedback, the team cannot
know if the problem is actually solved. Feedback drives iterative improvement
and model refinement. The methodology is cyclical — feedback triggers
revisions to earlier steps.

Q9. Comparative study: Train-Test Split vs Cross Validation.
Train-Test Split: large datasets, fast, one estimate, clear demarcation.
K-Fold CV: small datasets, slower (k models trained), more accurate
estimate, every point is used as test data at some point.

Q10. Why is model validation important?
Post-training, validation using test data measures accuracy on unseen data.
Benefits: enhances model quality, reduces error risk, prevents overfitting
(memorising training data) and underfitting (model too simple).

Q1. Explain k-fold cross validation with diagram.
Divide data into k equal folds (e.g., k=5, each fold = 20%).
Experiment 1: Fold 1 = Test, Folds 2-5 = Train → measure quality.
Experiment 2: Fold 2 = Test, Folds 1,3,4,5 = Train → measure quality.
...repeat for all k folds. Average the k quality measurements.
Result: 100% of data is used as holdout at some point.
Advantages: More reliable quality estimate than single split.
Disadvantage: Takes k times longer to run than train-test split.

Q2. Data requirements → collection → understanding → preparation.
Requirements (5W1H): Identify type (structured/unstructured), format,
 source, preprocessing needed.
Collection: Primary (surveys, IoT) or Secondary (Kaggle, WHO, web scrape).
 May be revisited after Understanding stage.
Understanding: Check data represents the problem. Apply descriptive stats
 (mean, std, correlation) and visualisation (histogram). Identify issues.
Preparation: Clean (fix NaN, typos, duplicates), combine sources, transform
 variables, feature engineering. MOST time-consuming step.

VSQ 3. What is a Confusion Matrix? Name its four components.
A Confusion Matrix is a table used to evaluate a classification model by comparing predicted outcomes against actual outcomes. For binary classification it is a 2×2 matrix with:

• TP (True Positive) — Predicted Yes, Actually Yes
• TN (True Negative) — Predicted No, Actually No
• FP (False Positive) — Predicted Yes, Actually No
• FN (False Negative) — Predicted No, Actually Yes
VSQ 4. What is the difference between primary and secondary data sources?
Primary Data Source: Original, firsthand data collected directly through observation, surveys, interviews, or IoT sensors. Raw and unbiased.
Examples: Feedback forms, marketing campaign data, IoT sensor readings.

Secondary Data Source: Pre-existing data already stored and ready for use. Collected from books, journals, websites, or databases.
Examples: Kaggle, WHO, World Bank Open Data, Google, UNICEF.
Section C — Short Answer Questions (3 · 3–4 marks)
SQ 1. Explain the four types of Data Analytics with examples.
1. Descriptive Analytics — Summarises past data to understand WHAT happened. Uses graphs, mean, median, mode. Example: Average exam marks; last year's sales trends.

2. Diagnostic Analytics — Analyses WHY things happened using root cause analysis and correlation. Example: Why did sales drop? Was it poor service or bad product?

3. Predictive Analytics — Uses historical data to predict FUTURE outcomes using regression/classification. Example: Forecasting next quarter's sales; predicting customer churn.

4. Prescriptive Analytics — Recommends the BEST ACTION to take based on predictions. Example: Optimise festival-season pricing strategy using past data analysis.
SQ 2. Explain K-Fold Cross Validation. How does it differ from Train-Test Split?
K-Fold Cross Validation: The dataset is divided into k equal folds. For k=5, each fold = 20%. In Experiment 1, Fold 1 is the test set and Folds 2–5 are training. In Experiment 2, Fold 2 is test. This repeats k times, so 100% of data is used as a holdout at some point. The k quality estimates are averaged for a final result.

Differences:
• Train-Test Split: for large datasets, fast (one model), clear separation of train/test
• K-Fold CV: for small datasets, more accurate estimate, slower (k models trained), every data point serves as test at some point
SQ 3. Given TP=40, TN=30, FP=8, FN=2 — calculate Precision, Recall, F1-Score and Accuracy.
Precision = TP/(TP+FP) = 40/(40+8) = 40/48 = 0.833 → 83.3%

Recall = TP/(TP+FN) = 40/(40+2) = 40/42 = 0.952 → 95.2%

F1-Score = 2 × (0.833 × 0.952) / (0.833 + 0.952) = 2 × 0.793 / 1.785 = 0.889 → 88.9%

Accuracy = (TP+TN)/(TP+FP+FN+TN) = (40+30)/(40+8+2+30) = 70/80 = 0.875 → 87.5%
