Section 4.1 – 4.3

Introduction to Orange Data Mining

Data Mining is the process of discovering trends, useful information, and patterns from large datasets. It involves analyzing and interpreting data to extract meaningful insights that aid in decision-making. Orange is a component-based visual programming software package designed for data visualisation, machine learning, data mining, and data analysis — all without writing a single line of code.

What is Orange?

Orange is an open-source tool where components are called widgets. Workflows are built by dragging, dropping, and connecting these widgets on a blank canvas. It supports Data Science, Computer Vision (CV), and Natural Language Processing (NLP) in one unified environment.

Section 4.5

Components of Orange

Orange has three main components that work together to create powerful data analysis pipelines: the Blank Canvas, Widgets, and Connectors. Understanding these building blocks is the foundation of everything in Orange.
🖥️

1. Blank Canvas

The workspace where you build analysis workflows by dragging and dropping widgets. You can rearrange and connect them to form a data processing pipeline from input to output.

🧩

2. Widgets

Graphical elements that perform specific tasks or operations on data — from loading a file to visualising scatter plots to running machine learning algorithms.

🔗

3. Connectors

Lines that link widgets together on the canvas. They represent the flow of data from one widget to another — output of one widget becomes input of the next.

The Iris Flower Story

Iris flowers come in three varieties — Iris Setosa, Iris Versicolor, and Iris Virginica. At first glance these varieties look similar, but they differ in the dimensions of their petals and sepals. We use Orange to load the iris dataset (150 samples), visualise it, and build a model that can tell the varieties apart.

Section 4.7.1.2

Evaluating the Classification Model

Once classified, it is crucial to assess how well the model performs. Orange provides powerful evaluation tools including accuracy, precision, recall, F1 score, and the Confusion Matrix — all using cross-validation for robust testing.

Key Evaluation Metrics

Accuracy

  • Percentage of correctly classified instances
  • Iris model achieves ~93% accuracy
  • Best for balanced datasets

Precision

  • Proportion of true positives among all predicted positives
  • Avoids false alarms
  • Important when FP is costly

Recall

  • Proportion of true positives correctly identified
  • Avoids missed detections
  • Important when FN is costly

F1 Score

  • Harmonic mean of Precision and Recall
  • Balances both metrics in one number
  • Best for imbalanced datasets

Cross-Validation Explained

Cross-validation (also called rotation estimation) includes resampling and sample-splitting methods that use different portions of data to test and train a model on different iterations. Orange uses 10-fold cross-validation — data is split into 10 subsets, trained 10 times (each time using a different subset as test data), and results are averaged for a robust evaluation.

Confusion Matrix

The Confusion Matrix shows a detailed breakdown of classifications — how many instances were correctly and incorrectly identified for each class.

TermAbbreviationMeaning
True PositiveTPCorrectly predicted as the target class
True NegativeTNCorrectly predicted as NOT the target class
False PositiveFPIncorrectly predicted as the target class (Type I Error)
False NegativeFNIncorrectly missed as the target class (Type II Error)
Insight: In our Iris model, Iris Setosa is identified perfectly (100%), but the model may occasionally confuse Versicolor and Virginica because their petal dimensions overlap slightly — this is exactly what the confusion matrix reveals.

Section 4.7.2

Computer Vision with Orange

Images represent a rich source of information. With Orange's Image Analytics add-on, we can convert images into numerical representations (embeddings) and apply machine learning to cluster or classify image data — no programming required.
⚠️ Add-On Required: Go to Options → Add-ons → Install "Image Analytics". Restart Orange to enable the new widgets in the widget panel.

Example: Clustering Dogs & Cats

We obtain a dataset containing images of dogs and cats, then use Orange to automatically cluster them into two groups based on their visual characteristics — without manually labelling any image.

1
Install Image Analytics Add-On

Options → Add-ons → Image Analytics → Install → Restart Orange.

2
Import Images

Drag "Import Images" widget onto canvas. Upload the dataset folder containing images of dogs and cats.

3

4

Connect "Image Embedding" widget to Import Images. This sends images to a remote server and computes numerical embeddings using a deep neural network trained on millions of real-life images.

7

Connect another "Image Viewer" to Hierarchical Clustering. Select areas of the dendrogram to see which images are grouped together. Dogs and cats will form distinct clusters!

Natural Language Processing with Orange

Natural Language Processing (NLP) helps machines understand and learn from written words. With Orange's Text add-on, we can load, visualise, preprocess, and analyse textual data to discover patterns and themes — all through a visual workflow.
⚠️ Add-On Required: Go to Options → Add-ons → Install "Text". Restart Orange to enable NLP widgets in the widget panel.

Install Text Add-On

Navigate to Options → Add-ons → Text → Install. Restart Orange Data Mining to activate.

Load or Create Textual Data

Drag the "Corpus" widget to load existing data. OR use "Create Corpus" widget to input your own text (articles, reviews, stories). Double-click to add text.

Short Answer Questions

SQ 1. What are the different components of the Orange Data Mining Tool? Describe each briefly.
Orange has three main components:

1. Blank Canvas: The workspace where you build analysis workflows. You drag and drop widgets here and connect them to form a data processing pipeline from input to output.

2. Widgets: Graphical elements that perform specific tasks on data. Examples include the File widget (loads data), Scatter Plot (visualises data), Tree (classification model), Test and Score (evaluates model), Image Embedding (converts images to numbers), and Word Cloud (shows word frequencies).

3. Connectors: Lines that link widgets together on the canvas. They represent the flow of data from one widget to another — the output of one widget becomes the input of the next.
SQ 2. Explain Cross Validation. Why is it used in model evaluation?
Cross Validation (also called rotation estimation) is a model evaluation technique that uses different portions of data to test and train a model across multiple iterations.

How it works (10-Fold CV): The data is split into 10 equal subsets. The model is trained 10 times — each time using 9 subsets for training and 1 different subset for testing. Results from all 10 iterations are averaged to get the final evaluation score.

Why it is used: Testing on training data would give falsely perfect results because the model has already "seen" that data. Cross-validation ensures the model is always tested on unseen data, giving a much more realistic and robust estimate of how it will perform in the real world.
SQ 3. What is the purpose of using the "Corpus Viewer" widget in Orange?
The Corpus Viewer widget in Orange NLP allows users to browse through textual data loaded in a corpus. Its key purpose is to:

• View the full text of each document in the corpus
• Search for specific words within the corpus
• Highlight the searched words wherever they appear in the text

It provides an easy way to explore and verify the textual data before further processing — essentially serving as an interactive reading and searching tool for the corpus.
SQ 4. How does the "Preprocess Text" widget contribute to text analysis in Orange Data Mining?
The "Preprocess Text" widget cleans and normalises raw text data before analysis. It performs several operations:

1. Lowercase conversion: All text converted to lowercase (e.g., "Turtle" → "turtle") for consistency.
2. Tokenisation: Splits text into individual words or tokens for processing.
3. Punctuation removal: Removes .,!@#$ and other special characters that don't carry meaning.
4. Stop word filtering: Removes common words like "the", "is", "and" that appear frequently but carry little meaning.
5. Stemming/Lemmatisation (optional): Reduces words to their root form (e.g., "running" → "run").

Result: The cleaned text leads to a more accurate and meaningful Word Cloud and subsequent analysis, because only semantically significant words remain.
